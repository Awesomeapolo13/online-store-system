# Архитектура

## Общие сведения

Проект является модульным монолитом. Состоит из:
- общего ядра Shared;
- модуля заказов Order;
- модуля уведомлений Notification.

Каждый модуль реализует гексагональную архитектуру. Состоит из слоев:
- Infrastructure;
- Application;
- Domain.

На слое домена реализуются:
- Entity;
- Value Object;
- Domain Events;
- Repository Interfaces (директория Repository);
- Domain Exceptions;
- Domain Enums.

На слое приложения реализуются:
- команды и их обработчики (Command);
- запросы и их обработчики (Query);
- обработчики событий (Event);
- юскейсы (UseCase)

На инфраструктурном слое реализуются:
- инструменты работы с хранилищами (Database);
- инструменты работы с HTTP (Http\Controller);
- реализации репозиториев.

## Создать новый эндпоинт

Для создания нового эндпоинта необходимо:
- создать контроллер в App\ModuleName\Infrastructure\Http\Controller;
- создать ДТО для запроса в App\ModuleName\Application\Request;
- создать юскейс для обработки эндпоинта App\ModuleName\Application\UseCase;
- для обработки при необходимости создаются команды (Command), запросы (Query) и события (Event);
- если по итогу обработки нужно отдать данные в ответе, то создается класс асемблер App\ModuleName\Application\Assembler, который собирает ответ в ДТО ответа App\ModuleName\Application\Response;
- сбор ответа в JSON реализуется посредством symfony/serializer. 

Контроллеры расположены в директориях своих модулей на слое инфраструктуры, например, `App\Order\Infrastructure\Http\Controller\GetCartController`. 
Каждый контроллер представляет собой класс с единственным методом `_invoke` и ДТО запроса в нем. Сам ДТО реализуется в Application слое. 
Маппинг в ДТО осуществляется посредством функционала Symfony. Контроллер передает запрос в юскейс, реализованный на уровне приложения.

Юскейс так же является invokable классом в котором осуществляется обработка эндпоинта. В зависимости от того, что требуется в эндпоинте, там 
вызывается либо шина команд, либо запросов.
